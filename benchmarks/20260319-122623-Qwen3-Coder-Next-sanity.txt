RESPONSE: The **binary chop algorithm**, more commonly known today as **binary search**, is a highly efficient algorithm for finding a target value within a **sorted** collection (typically an array or list). Its name “binary chop” emphasizes its divide-and-conquer nature: it repeatedly *chops* the search space in half until the target is found—or determined to be absent.

Below is a comprehensive, in-depth explanation covering principles, mechanics, correctness, complexity, variations, and practical considerations.

---

## 🔍 Core Idea

Given a **sorted** array `A` of length `n`, and a target value `T`, binary search works by:

1. Comparing `T` to the **middle element** of the current search interval.
2. If `T` equals the middle element → **found**.
3. If `T` is less than the middle element → **discard the right half** (since the array is sorted).
4. If `T` is greater than the middle element → **discard the left half**.
5. Repeat the process on the remaining half.

This halving continues until the interval is empty (target not found) or the target is located.

fn quicksort<T: Ord + Clone>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    let pivot_index = partition(arr);
    let (left, right) = arr.split_at_mut(pivot_index);
    quicksort(left);
    quicksort(&mut right[1..]); // Skip the pivot element
}
