RESPONSE: The **binary chop algorithm**, more commonly known as **binary search**, is a highly efficient algorithm for finding a target value within a **sorted** collection (typically an array or list). Its name “binary chop” emphasizes its divide-and-conquer nature: it repeatedly *chops* the search space in half until the target is found or the space is exhausted.

Below is a comprehensive, in-depth explanation covering principles, mechanics, correctness, complexity, variations, and practical considerations.

---

## 🔍 Core Idea

Given a **sorted** sequence (ascending or descending), binary search exploits ordering to eliminate half of the remaining candidates at each step. Instead of scanning linearly (O(n)), it reduces the problem size logarithmically.

### High-Level Strategy
1. Compare the target value to the **middle element** of the current search interval.
2. If equal → found.
3. If target < middle → discard the right half (search left).
4. If target > middle → discard the left half (search right).
5. Repeat on the remaining half.

RESPONSE: Here's a clean, idiomatic in-place quicksort implementation in Rust, using the Lomuto partition scheme:

```rust
fn quicksort<T: Ord + Clone>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    let pivot_index = partition(arr);
    let (left, right) = arr.split_at_mut(pivot_index);
    
    quicksort(left);
    quicksort(&mut right[1..]); // Skip the pivot element
}

fn partition<T: Ord + Clone>(arr: &mut [T]) -> usize {
    let len = arr.len();
    let pivot_index = len - 1;
    let mut i = 0;

    for j in 0..pivot_index {
        if arr[j] <= arr[pivot_index] {
            arr.swap(i, j);
            i += 1;
        }
    }
    arr.swap(i, pivot_index);
    i
}
```
