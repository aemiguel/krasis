## 🧠 Core Principle

Binary search leverages the **sorted nature** of the input array. Instead of scanning all elements sequentially (as in linear search), it compares the target value to the **middle element** of the current search range:

- If the target equals the middle element → **found**.
- If the target is *less than* the middle element → the target (if present) must be in the **left half** → recurse/iterate on the left half.
- If the target is *greater than* the middle element → the target must be in the **right half** → recurse/iterate on the right half.

This “divide and conquer” strategy reduces the problem size by half with each step.
