PROMPT: Describe the binary chop algorithm in depth
RESPONSE: The **binary chop algorithm**—more commonly known as **binary search**—is an efficient, divide-and-conquer algorithm for finding a target value within a **sorted** array (or list). Its name "binary chop" emphasizes the repeated halving (or "chopping") of the search space. Below is a comprehensive, in-depth explanation covering its principles, mechanics, variants, complexity, implementation, and nuances.

---

## 🧠 Core Concept

Binary search works by **repeatedly dividing the search interval in half**. It starts with the entire sorted array, compares the target value to the middle element, and:

- If they match, the search ends successfully.
- If the target is *less than* the middle element, the search continues in the **left half**.
- If the target is *greater than* the middle element, the search continues in the **right half**.

This process repeats until the target is found or the search interval becomes empty (indicating absence).

> ⚠️ **Crucial Requirement**: The input array **must be sorted** (e.g., ascending order). Unsorted data renders binary search invalid or requires preprocessing (e.g., sorting first), which often negates its efficiency advantage.

pub fn quicksort<T: Ord + Copy>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    // Partition the array
    let pivot = arr[arr.len() / 2]; // middle element as pivot
    let mut i = 0;
    let mut j = arr.len() - 1;

    loop {
        // Find element >= pivot from left
        while i < j && arr[i] < pivot {
            i += 1;
        }

        // Find element <= pivot from right
        while j > i && arr[j] > pivot {
            j -= 1;
        }

        if i >= j {
            break;
        }

        arr.swap(i, j);
        i += 1;
        j -= 1;
    }

    // Recursively sort the partitions
    // Note: j is the last index of the left partition
    let left_size = j + 1;
    quicksort(&mut arr[0..left_size]);
