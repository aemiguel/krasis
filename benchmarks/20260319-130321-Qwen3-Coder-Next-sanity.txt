PROMPT: Describe the binary chop algorithm in depth
RESPONSE: The **Binary Chop** (more commonly known as **Binary Search**) is a highly efficient algorithm for finding a target value within a **sorted** array. It works by repeatedly dividing the search interval in half, leveraging the sorted order to eliminate large portions of the data with each comparison. Its time complexity is **O(log n)**—dramatically faster than linear search’s O(n) for large datasets.

 Below is a comprehensive, in-depth explanation:

---

### **1. Core Principle**
Binary search operates on the principle of **divide-and-conquer**:
- At each step, it compares the target value to the **middle element** of the current search interval.
- If they are unequal, the algorithm determines whether the target lies in the **left half** (if target < middle) or **right half** (if target > middle).
- It then **discards** the half where the target cannot exist and continues with the remaining half.
- This process repeats until the target is found or the interval becomes empty (indicating absence).

pub fn quicksort<T: Ord + Clone>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    // Hoare partition scheme
    fn partition<T: Ord + Clone>(arr: &mut [T], low: usize, high: usize) -> usize {
        let pivot = arr[low].clone(); // choose first element as pivot
        let mut i = low.wrapping_sub(1); // start just before low
        let mut j = high + 1;            // start just after high

        loop {
            // Find element >= pivot from left
            loop {
                i = i.wrapping_add(1);
                if arr[i] >= pivot { break; }
            }

            // Find element <= pivot from right
            loop {
                j = j.wrapping_sub(1);
                if arr[j] <= pivot { break; }
            }

            if i >= j {
                return j;
            }

            arr.swap(i, j);
        }
