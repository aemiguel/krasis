## 🔍 Core Idea

Given a **sorted** sequence (e.g., ascending order), binary search exploits the ordering to eliminate **half** of the remaining elements at each step:

1. Compare the target value to the **middle element** of the current search interval.
2. If equal → found.
3. If target < middle → discard the right half (search left).
4. If target > middle → discard the left half (search right).
5. Repeat on the remaining half.

This divide-and-conquer strategy yields logarithmic time complexity.

fn quicksort<T: Ord + Copy>(arr: &mut [T]) {
    if arr.len() <= 1 {
        return;
    }

    // Partition the array around a pivot (middle element)
    let pivot_index = partition(arr);
    
    // Recursively sort the two partitions
    quicksort(&mut arr[..pivot_index]);
    quicksort(&mut arr[pivot_index + 1..]);
}

    // Choose middle element as pivot to avoid worst-case on sorted arrays
    let pivot_index = arr.len() / 2;
