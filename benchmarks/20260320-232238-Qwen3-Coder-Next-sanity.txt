### 1. Core Principle & Intuition

Binary search exploits the **ordering** of the data. At each step, it compares the target value to the *middle element* of the current search interval:

- If they match → found!
- If the target is smaller → search the *left half* (since the array is sorted ascending).
- If larger → search the *right half*.

This “divide and conquer” strategy reduces the problem size by half at each step, leading to logarithmic time complexity.
